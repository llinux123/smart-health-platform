# 项目启动说明

**项目名称：** 智慧医疗与大健康管理平台  
**技术栈：** Spring Boot 3.2.5 + Spring Cloud Alibaba + Vue 3 + Vite 5

---

## 1. 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 17+ |
| Maven | 3.8+ |
| Node.js | 18+ |
| Docker & Docker Compose | 最新 |

---

## 2. 启动基础设施（Docker）

使用 Docker Compose 一键启动 MySQL、Redis、RabbitMQ、Elasticsearch、Nacos：

```bash
docker compose up -d
```

等待所有容器启动完毕（约 1-2 分钟），可通过以下方式验证：

| 服务 | 地址 | 验证方式 |
|------|------|----------|
| MySQL | `localhost:3306` | `docker logs smart-health-mysql` |
| Redis | `localhost:6379` | `docker logs smart-health-redis` |
| RabbitMQ | `localhost:15672` | 访问 http://localhost:15672（guest/guest） |
| Elasticsearch | `localhost:9200` | `curl http://localhost:9200` |
| Nacos | `localhost:8848` | 访问 http://localhost:8848/nacos（nacos/nacos） |

> 首次启动时 MySQL 会自动执行 `sql/init.sql` 初始化数据库表结构和种子数据。
> Nacos 控制台访问路径为 `http://localhost:8848/nacos/`。

## 3. 启动后端服务

### 3.1 编译

```bash
mvn clean package -DskipTests
```

### 3.2 启动方式

后端为微服务架构，提供两种启动方式：

**方式一：Docker Compose（推荐）**

使用 Docker Compose 一键启动所有微服务（含基础设施）：

```bash
docker compose up -d --build
```

此方式会自动构建并启动以下服务：

| 服务 | 端口 | Nacos 注册名 |
|------|------|-------------|
| smart-health-user | 8081 | `user-service` |
| smart-health-registration | 8082 | `registration-service` |
| smart-health-consultation | 8083 | `consultation-service` |
| smart-health-prescription | 8084 | `prescription-service` |
| smart-gateway | 9000 | `gateway-service` |

各微服务启动后自动注册到 Nacos，通过 Gateway（端口 9000）统一对外暴露 API。

**方式二：逐一启动各微服务（本地开发）**

确保基础设施已启动后，按以下顺序逐一启动：

```bash
# 启动网关（端口 9000）
java -jar smart-gateway/target/smart-gateway-1.0.0-SNAPSHOT.jar

# 启动用户/认证服务（端口 8081）
java -jar smart-health-user/target/smart-health-user-1.0.0-SNAPSHOT.jar

# 启动挂号/排班服务（端口 8082）
java -jar smart-health-registration/target/smart-health-registration-1.0.0-SNAPSHOT.jar

# 启动 AI 问诊服务（端口 8083）
java -jar smart-health-consultation/target/smart-health-consultation-1.0.0-SNAPSHOT.jar

# 启动处方/药房服务（端口 8084）
java -jar smart-health-prescription/target/smart-health-prescription-1.0.0-SNAPSHOT.jar
```

> 各服务启动后会自动注册到 Nacos。建议先启动 Gateway，再启动其他服务。

### 3.3 使用 IntelliJ IDEA 启动

项目已预配置所有 Spring Boot 启动类的 Run Configuration，可直接在 IDEA 中使用。

**前置步骤：导入项目**

1. 打开 IDEA，选择 **File → Open**，选择项目根目录
2. 确保 **Project SDK** 设置为 JDK 17（**File → Project Structure → Project → SDK**）
3. 等待 Maven 依赖下载完成（右下角进度条结束）

**方式一：聚合启动（推荐）**

在 IDEA 右上角的运行配置下拉框中，选择 **SmartHealthApplication**，点击绿色运行按钮即可启动聚合服务（端口 8080）。再切换至 **GatewayApplication** 配置启动网关（端口 9000）。

**方式二：逐一启动各微服务**

项目预置了以下 Spring Boot 运行配置，按需逐一启动：

| 运行配置名称 | 模块 | 启动类 | 端口 |
|-------------|------|--------|------|
| GatewayApplication | smart-gateway | `com.smart.health.gateway.GatewayApplication` | 9000 |
| UserApplication | smart-health-user | `com.smart.health.user.UserApplication` | 8081 |
| RegistrationApplication | smart-health-registration | `com.smart.health.registration.RegistrationApplication` | 8082 |
| ConsultationApplication | smart-health-consultation | `com.smart.health.consultation.ConsultationApplication` | 8083 |
| PrescriptionApplication | smart-health-prescription | `com.smart.health.prescription.PrescriptionApplication` | 8084 |

```bash
# 或使用 Maven 分别启动各模块
mvn spring-boot:run -pl smart-gateway
mvn spring-boot:run -pl smart-health-user
mvn spring-boot:run -pl smart-health-registration
mvn spring-boot:run -pl smart-health-consultation
mvn spring-boot:run -pl smart-health-prescription
```

> 需要在 IDEA 中运行 `SmartHealthApplication`（smart-health-app 模块）聚合启动时，该模块已预配置运行配置，可一键启动所有业务模块（端口 8080），但微服务模式下建议逐一启动各独立服务。

**配置环境变量（可选）**

若需要 AI 问诊功能，需在各运行配置中添加环境变量：

1. 点击 **Run → Edit Configurations**
2. 选择目标运行配置
3. 在 **Environment variables** 栏添加：
   - `ZHIPU_API_KEY=你的智谱API Key`
   - `AI_API_KEY=你的百炼API Key`

> 也可直接在项目根目录的 `.env` 文件中配置，项目通过 `spring-dotenv` 自动加载。

## 4. 启动前端

```bash
cd smart-health-frontend
npm install
npm run dev
```

前端默认运行在 `http://localhost:5173`，已配置代理将 `/api` 请求转发至 Gateway（`http://localhost:9000`）。

若使用 Docker Compose 启动前端，则前端运行在 `http://localhost:5173`（Nginx 反向代理至 Gateway）。

## 5. 访问入口

| 入口 | 地址 | 说明 |
|------|------|------|
| 前端页面（Docker） | http://localhost:5173 | Nginx + Vue SPA，反向代理至 Gateway |
| 前端页面（Vite Dev） | http://localhost:5173 | Vite 开发服务器，热重载，代理至 Gateway |

| Nacos 控制台 | http://localhost:8848/nacos/ | 服务注册与发现（默认无认证） |
| RabbitMQ 控制台 | http://localhost:15672 | 消息队列管理（guest/guest） |

## 5.1 E2E 测试

### 前置条件

- 前端 Vite dev server 运行在 `http://localhost:5173`
- 后端 Gateway 运行在 `http://localhost:9000`（且所有微服务已注册到 Nacos）
- Playwright 已安装（`cd smart-health-frontend && npx playwright install chromium`）

### 运行测试

```bash
cd smart-health-frontend

# 运行所有 E2E 测试
npx playwright test

# 运行指定测试文件
npx playwright test e2e/consultation-handoff.spec.ts

# 带 UI 交互模式
npx playwright test --ui
```

### 测试账号

| 用户类型 | 用户名 | 密码 |
|---------|--------|------|
| 患者 | `testpatient` | `123456` |
| 医生 | `doc_wangminghua` | `doctor123` |

### 常见问题

**1. Playwright 报错 EROFS: read-only file system**

在 Linux sandbox 环境中，Playwright 默认使用 `/tmp` 写入缓存，但 `/tmp` 可能为只读。通过环境变量指定可写目录：

```bash
TMPDIR=/path/to/project/.playwright-tmp \
PLAYWRIGHT_TMP_DIR=/path/to/project/.playwright-tmp \
npx playwright test
```

## 6. 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `ZHIPU_API_KEY` | 智谱 AI API Key（多模态图片分析） | — |
| `AI_API_KEY` | 阿里云百炼 API Key（Qwen 对话） | — |
| `NACOS_SERVER_ADDR` | Nacos 注册中心地址 | `localhost:8848` |
| `JWT_SECRET` | JWT 签名密钥 | 内置默认值 |

配置方式：通过 `export VARIABLE_NAME=value` 设置环境变量。
> 注意：`.env` 文件（项目根目录）仅含 `ZHIPU_API_KEY` 和 `AI_API_KEY`，其他变量需通过环境变量或 JVM 参数设置。
部分模块依赖 `spring-dotenv` 加载 `.env` 文件，若变量未生效请改用 `export` 方式。

## 7. 快速启动（全量 Docker Compose）

```bash
# 1. 编译后端所有模块
mvn clean package -DskipTests

# 2. 构建前端
cd smart-health-frontend && npm ci && npm run build && cd ..

# 3. 一键启动全部服务（基础设施 + 后端微服务 + 网关 + 前端）
docker compose up -d --build

# 4. 查看服务状态
docker compose ps

# 5. 查看日志
docker compose logs -f
```

启动后访问 http://localhost:5173 即可使用。

## 8. 常见问题

### 8.1 Docker 前端端口无法访问

若前端容器运行但端口映射未生效（`docker ps` 显示 `80/tcp` 而非 `0.0.0.0:5173->80/tcp`），说明特权端口（<1024）绑定失败。
修改 `docker-compose.yml` 中前端端口映射为 `5173:80`，或使用其他非特权端口。

### 8.2 Elasticsearch 容器因内存不足退出

调整 `docker-compose.yml` 中的 `ES_JAVA_OPTS`，或增加 Docker 内存限制。

### 8.3 Nacos 连接失败

确保 Nacos 容器已完全启动（日志中出现 `Nacos started successfully`），且后端服务配置中的 Nacos 地址正确。
Nacos Web 控制台访问路径为 `http://localhost:8848/nacos/`（带 `/nacos` 上下文路径），Spring Cloud 客户端无需添加此路径。

### 8.4 前端代理不生效

检查 `smart-health-frontend/vite.config.ts` 中 proxy 的 target 是否指向正确的 Gateway 地址（默认 `http://localhost:9000`）。
若使用 Docker Compose 启动，Nginx 已内置反向代理配置，无需 Vite proxy。

### 8.5 AI 接口返回 403

检查 `.env` 文件中的 `ZHIPU_API_KEY` 和 `AI_API_KEY` 是否配置正确且未过期。

### 8.6 Gateway 启动 StackOverflowError

在微服务模式下，Gateway 的 `spring-boot-maven-plugin` 已移除 `exec` classifier（JAR 名不再带 `-exec` 后缀），
解决了 `StackOverflowError` 问题。若使用旧版 JAR，请重新执行 `mvn clean package -DskipTests` 编译。
