# 智慧医疗平台 - 预置账号参考手册

> 本文档为开发/测试环境的预置账号手册，**请勿在生产环境中使用**。

---

## 登录方式

- **患者端**：http://localhost:8080（前端页面）
- **员工端（医生/药师/运维）**：HTTP POST `http://localhost:8080/api/v1/auth/login`
  - 请求体：`{ "loginType": "STAFF", "username": "xxx", "password": "xxx" }`
- 员工登录后，`LoginResponse.role` 字段表明角色，前端据此展示对应管理界面。

| 真实姓名 | 系统管理员 |
| 角色 | ADMIN |

| 状态 | **正常可用** |
| 来源 | V3 迁移脚本预置 |

### 验证 admin 是否可用的 SQL

```sql
SELECT id, username, real_name, role, is_deleted, create_time
FROM t_staff
WHERE username = 'admin';
```

期望结果：返回一条 `is_deleted = 0` 的记录，表明 admin 账号正常。
